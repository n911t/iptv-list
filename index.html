📘 كراس المطور: الدليل الشامل للهندسة العكسية

دليل متكامل لفحص تطبيقات الأندرويد واستخراج الروابط برمجياً باستخدام (Frida & ADB)

🗺️ المخطط المعماري للعملية

قبل البدء بالخطوات، يجب فهم الخريطة الذهنية للعملية والتي تعتمد على نظامين منفصلين يعملان معاً:

[ نظام الويندوز - Windows ] <========== مسار الاتصال (ADB) ==========> [ المحاكي - Android ]
       |                                                               |
(CMD 2) العميل: أدوات فريدا                                      (CMD 1) الخادم: نظام الأندرويد
يُرسل أمر المراقبة ويطبع النتيجة                                 يستضيف التطبيق المستهدف
       |                                                               |
[ سكربت الفخ .js ] -----------------(حقن الكود الديناميكي)-----------------> [ ذاكرة التطبيق (RAM) ]

🛠️ الفصل الأول: تجهيز مسرح العمليات (إعداد المحاكي)

نظام الأندرويد العادي يشبه القلعة المغلقة. لكسر هذه الحماية، نستخدم "المحاكي" ونمنح أنفسنا الصلاحيات المطلوبة.

  1. تفعيل صلاحيات الجذر (Root Permission):

    الدخول لإعدادات المحاكي > إعدادات النظام > تفعيل Root Permission. هذا يعطينا "المفتاح الذهبي" للوصول لملفات النظام العميقة وتعديلها.

  2. تفعيل وضع تصحيح الأخطاء (USB Debugging):

    من داخل الأندرويد: الإعدادات > حول الجهاز > اضغط 7 مرات على رقم الإصدار. ثم ارجع للخلف > خيارات المطور > فعّل تصحيح أخطاء USB. هذا يفتح باباً يسمح للويندوز بإرسال أوامر برمجية مباشرة لعقل الأندرويد.

🌉 الفصل الثاني: مد جسر التواصل (أداة ADB)

هو الكابل الافتراضي الذي ينقل الأوامر والملفات بين الويندوز والأندرويد.

  • توجيه شاشة الأوامر: افتح نافذة CMD جديدة واكتب: 
    cd C:\LDPlayer\LDPlayer9
  • فحص الاتصال: للتأكد من أن الويندوز يرى المحاكي: 
    adb devices

🕵️‍♂️ الفصل الثالث: استراتيجية الشاشتين وزراعة الجاسوس

أداة Frida تعمل بنظام (خادم / عميل)، لذا نحتاج إلى نافذتي CMD منفصلتين للتحكم.

النافذة الأولى (CMD 1): "غرفة تحكم الأندرويد" (The Server)

هذه النافذة ستتحول من نظام الويندوز إلى شاشة تحكم لينكس/أندرويد.

  1. نقل ملف الجاسوس (فريدا) للأندرويد: 
    adb push frida-server /data/local/tmp/
  2. الدخول إلى بيئة الأندرويد: 
    adb shell
  3. الترقية إلى المستخدم الخارق (Super User): 
    su
    💡 معلومة معمارية: الأمر su ينقلنا من مستخدم عادي مقيد بصلاحيات القراءة، إلى مدير مطلق. لولا تفعيلنا للروت في الفصل الأول، لرفض النظام هذا الأمر.
  4. إعطاء الصلاحيات وتشغيل الجاسوس في الخلفية: 
    chmod 777 /data/local/tmp/frida-server
/data/local/tmp/frida-server &
⚠️ تنبيه: يجب ترك هذه النافذة مفتوحة وعدم إغلاقها إطلاقاً حتى انتهاء العمل، لأن إغلاقها يقتل الخادم.

🕸️ الفصل الرابع: كتابة الفخ (سكربت الجافاسكربت)

نحتاج الآن إلى كتابة الكود الذي سيراقب الذاكرة ويصطاد الرابط المباشر.

  1. افتح برنامج المفكرة (Notepad).
  2. انسخ هذا الكود والصقه: 
    Java.perform(function () {
    console.log("[*] جاري مراقبة مشغل الفيديو...");
    
    var MediaPlayer = Java.use("android.media.MediaPlayer");
    
    MediaPlayer.setDataSource.overload("java.lang.String").implementation = function (path) {
        console.log("[+] الرابط المكتشف هو: " + path);
        return this.setDataSource(path); // السماح للرابط بالمرور حتى لا يتعطل التطبيق
    };
});
  3. احفظ الملف باسم oscar-test.js مباشرة في القرص الصلب C:\ (لتجنب أخطاء مسارات الويندوز الطويلة).

🎯 الفصل الخامس: لحظة الصيد والاستخراج

الآن، نعود لنظام الويندوز لكي نتواصل مع الخادم الذي تركناه يعمل في الخلفية.

النافذة الثانية (CMD 2): "محطة الاستقبال في الويندوز" (The Client)

  1. تجهيز التطبيق: افتح التطبيق المستهدف داخل المحاكي واجعله نشطاً على الشاشة (Foreground).
  2. تنفيذ أمر الربط (Attaching): افتح شاشة CMD جديدة كلياً واكتب الأمر التالي: 
    %APPDATA%\Python\Python314\Scripts\frida -U -F -l C:\oscar-test.js
  3. تحليل الأمر:
    • -U : الاتصال عبر منفذ الـ USB (المحاكي).
    • -F : الارتباط فوراً بالتطبيق المعروض على الشاشة دون الحاجة لمعرفة اسم الحزمة.
    • -l : تحميل سكربت الفخ وحقنه داخل الذاكرة الحية للتطبيق.
✅ النتيجة المتوقعة: بمجرد تشغيل أي قناة داخل المحاكي، سيتدخل السكربت، يوقف الرابط، ويطبعه بنص صريح على شاشة الويندوز، ثم يسمح للمشغل بإكمال عمله.

💡 دروس مستفادة للمطورين (Architectural Takeaways)

هذه التجربة تثبت حقيقة أمنية بالغة الأهمية: تمرير الروابط أو البيانات الحساسة بين مكونات التطبيق بنص صريح (Plain Text) يعتبر ثغرة أمنية فادحة.

لحماية التطبيقات مستقبلاً، يجب استخدام التقنيات التالية:

  • التشفير (Encryption): إرسال الروابط مشفرة برمجياً وفك تشفيرها مؤقتاً داخل المشغل نفسه.
  • الرموز المؤقتة (Tokenization): حماية روابط الخوادم برموز تحقق تتغير زمنياً (Tokens) لمنع تشغيلها خارج نطاق التطبيق الرسمي.
  • التحقق من الهوية (User-Agent Validation): رفض أي اتصال بالخادم لا يحمل بصمة التطبيق الخاص بك.
